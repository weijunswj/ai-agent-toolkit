# Baseline Workflow

Use this for broad, ambiguous, high-blast-radius, or multi-step tasks.

## Steps

- Restate the concrete goal in your own working notes.
- Inspect only the files and docs needed to understand the change.
- Identify the smallest safe edit and the smallest meaningful validation.
- Prefer existing project patterns over new abstractions.
- Keep unrelated refactors out of the diff.
- Report blockers early when safety, missing context, or conflicting instructions prevent progress.

For small clear tasks, this playbook is optional; proceed with `AGENTS.md` and targeted repo context.
