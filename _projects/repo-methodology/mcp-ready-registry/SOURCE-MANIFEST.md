# Source Manifest: MCP-Ready Registry

## Preserved In `_main/`

- `README.md`
- `installer-mcp/**`
- `registry-mcp/**`
- `references/**`
- `registry/**`

This module owns repo-level MCP-ready registry docs/specs and registry metadata. The source material is first-party toolkit-authored content promoted from the former manually maintained `mcp/` published surface.

## Published MCP Surfaces

- `mcp/README.md` describes the current MCP-ready registry surface and states that no runnable MCP server is shipped.
- `mcp/registry/**` is machine-readable metadata for future discovery and installer flows.
- `mcp/registry-mcp/**` is future design/spec-only documentation for read-only registry discovery.
- `mcp/installer-mcp/**` is future design/spec-only documentation for approval-gated pack installer behavior.
- `mcp/references/**` is supporting operator and design reference material.

Project-specific MCP notes under `mcp/projects/**` remain owned by each project module and are out of scope for this module. `mcp/registry/projects.registry.json` remains generated by toolkit project sync from all project manifests.

## Runtime Status

This is MCP-ready registry and design/spec material. It is not a runnable MCP server.

This module must not add an npm package, CLI, server entrypoint, live MCP runtime, network service, API integration, executable MCP tools, credentials, `.env*`, `.tmp/`, `.n8n-local/`, or live n8n actions.

## Link Shims

No compatibility shims are used in this module.

## Excluded

- Runnable MCP server code, CLI code, API clients, package manifests, network services, or executable tools.
- Credentials, `.env*`, private keys, credential exports, live imports/exports, package artifacts, and product repo files.
- Project-specific MCP notes under `mcp/projects/**`.
- `mcp/registry/projects.registry.json`, which is generated from project manifests by sync.
