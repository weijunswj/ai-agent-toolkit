The local stack templates under `_main/templates/.n8n-local/` are toolkit-authored source for the local `n8n + postgres` Fast Path, with the Compose ngrok service as the supported local public tunnel path.

The production Cloudflare stack templates under `_main/templates/.n8n-production-cloudflare/` are toolkit-authored source for production self-hosting from a local/CGNAT machine through Cloudflare Tunnel. This stack is intentionally separate from the local dev stack and keeps Postgres private, avoids public direct n8n host port `5678`, and uses `cloudflared` with a tunnel token from private environment.

The production server backup templates under `_main/templates/production-server-backups/` are toolkit-authored source for Linux server deployments such as Hostinger Coolify VPS or a company server. They are intentionally separate from the local Windows launcher and use server-appropriate systemd timer or cron scheduling guidance instead of Windows Task Scheduler.

## AI-Facing Surfaces

- `skills/n8n-local-setup/references/n8n/local-setup.md`, `skills/n8n-local-setup/references/n8n/hostinger-vps.md`, and `skills/n8n-local-setup/references/n8n/production-cloudflare-tunnel.md` are full guide copies generated from `_main/`. The hosted guide covers Hostinger Coolify VPS n8n deployment after Coolify exists, including Coolify routing, private Postgres, and a healthchecked starter Compose. The production Cloudflare guide covers local/CGNAT production self-hosting with an outbound Cloudflare Tunnel connector.
- `skills/n8n-local-setup/references/ai-agent-platforms/codex.md`, `claude-code.md`, `opencode.md`, and `antigravity.md` are [official n8n Skills](https://github.com/n8n-io/skills) plus instance-level MCP setup guide copies generated from the confirmed source pages in `_main/`.
- `skills/n8n-local-setup/templates/mcp-configs/*-mcp-config.md` are official instance-level MCP config templates generated by copy recipes from the confirmed config sources in `_main/templates/mcp-configs/`.
- `skills/n8n-local-setup/templates/.n8n-local/docker-compose.yml`, `.env.example`, `.gitignore`, `_n8n-local.cmd`, `n8n-local-desktop-shortcut.cmd`, and `scripts/n8n-local-menu.ps1` are generated by exact copy recipes from `_main/templates/.n8n-local/`. The menu and CMD wrapper use fail-closed Docker CLI, Compose, and engine preflight before drawing the main screen. Missing CLI or Compose produces official manual Docker Desktop guidance and a clean one-Enter exit; an already-installed stopped Docker Desktop can still be started with a bounded readiness wait. The launcher does not install Docker Desktop, invoke a package manager, or open the documentation URL. The stack also includes local-only safe n8n CLI backup zip packages, automatic backup setup/removal, manifests, and retention cleanup.
- `skills/n8n-local-setup/templates/.n8n-production-cloudflare/docker-compose.yml`, `.env.example`, `.gitignore`, `_n8n-production-cloudflare.cmd`, and `scripts/n8n-production-cloudflare-menu.ps1` are generated by exact copy recipes from `_main/templates/.n8n-production-cloudflare/`. The production menu and CMD wrapper use the same manual Docker prerequisite guidance and installed-Docker startup preflight as the local stack. Production n8n start, restart, recreate, update, self-heal, and restore restart paths also require stable configured-loopback readiness or an honest bounded timeout with redacted recent logs; stop/down paths remain unwaited and tunnel health stays distinct.
- `skills/n8n-local-setup/templates/production-server-backups/README.md` and `n8n-production-backup.sh.template` are generated by exact copy recipes from `_main/templates/production-server-backups/`. They document and implement Linux production backups with n8n CLI workflow/credential exports, decrypted credential export disabled by default, Postgres backup where applicable, timestamped backup folders, manifest, restore notes, failure logging/status, retention cleanup, and systemd timer or cron scheduling guidance.
- AI-facing skill, n8n/platform indexes, and pack surfaces are generated from `curated_output_for_ai/`.
- `skills/n8n-local-setup/agents/openai.yaml` is generated from reviewed curated OpenAI invocation-policy metadata.
- Full n8n operating rules are owned by `_projects/development/ai-coding-agent-rules/_main/_partials/n8n-agent-rules.md` and published through `skills/n8n-agent-rules/`.
- `skills/n8n-local-setup/references/n8n-agent-rules.md` is a generated cross-skill reference copied from the canonical `n8n-agent-rules` source for portability. It is labelled as generated cross-skill content and must not be edited directly.

## Collapsed Or Retired

- Standalone update, tunnel, Compose appendix, old platform-integration, Hostinger one-click n8n template guidance, and generic hosting guide pages were collapsed, de-scoped, or replaced by the current main pages and skills-first platform routers.
- [Official n8n Skills](https://github.com/n8n-io/skills) plus instance-level MCP references are secondary skill surfaces. They are not part of the beginner local setup flow.
- The legacy Windows helper BAT file was removed. `_n8n-local.cmd` is the supported launcher.
- Small compatibility shims for old numbered guide links were removed because the final generated surface now uses clean current paths.

## Excluded

- Credentials, `.env*` except placeholder-only `.env.example`, `.n8n-local/`, `.tmp/`, live n8n exports/imports, package artifacts, and private product files.
- Live workflow state. This toolkit module preserves setup documentation and placeholder-only templates only.
- Real Cloudflare tunnel tokens, account IDs, DNS values, domains, IPs, certificates, backups, production `.env` values, and private deployment notes.
- Full n8n agent operating rules. Use `skills/n8n-agent-rules/` for that contract.
