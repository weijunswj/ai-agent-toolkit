# External System Router

First-party Toolkit safety router for owner-authorised external-system operations.

It provides:

- Tier 0-3 risk classification and task-authorisation envelopes.
- An unconditional Tier 2 floor for every production mutation plus task/session/objective-bound task lifetimes.
- Canonical versioned operation semantics; caller read/write labels are assertions and never establish mutability.
- A bounded canonical inventory loader with process-local authority, source/topology/runtime identity checks, monotonic generations, and unforgeable host-plan derivation.
- Operation-specific interface audits and route selection without a global MCP/API/browser hierarchy.
- Universal graphical-control disclosure and last-resort browser-history approval.
- Repository-and-intent capability reconciliation, dynamic domain markers, and component-based n8n recommendations.
- Mandatory fail-closed n8n operation classification, exact official-Skill attestation, per-task ledgers, Claude Toolkit-direct mutation denial, and completion auditing without claiming unproven Codex hook parity.
- A host-neutral authenticated provider target registry plus loader-derived Codex and Claude adapter plans required by final route and receipt creation.
- Consumer requirement, target registry, capability audit/ledger, and operation receipt schemas.
- Staged promotion/demotion and deterministic drift evidence contracts.

This skill does not grant live-system authority and is not a generic HTTP, shell, MCP, connector, or browser executor.
