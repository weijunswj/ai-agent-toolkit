# Incident Response Checklist

- [ ] Pending - Preserve evidence first; do not delete logs.
- [ ] Pending - Identify affected service, symptoms, timeline, and current public exposure.
- [!] Needs owner approval - Contain public exposure, restart services, reboot, rollback, restore, or perform destructive actions unless pre-approved by an incident runbook.
- [ ] Pending - Confirm backup state.
- [ ] Pending - Write incident report and follow-up tasks.
- [x?] Completed but needs human verification - Owner verifies customer/business impact and external communications.
